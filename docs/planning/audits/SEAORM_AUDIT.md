# SeaORM Feature Audit for Epic 02

This document audits all SeaORM ORM objects and features to ensure Epic 02 replicates them all.

## SeaORM Core Components

### 1. Entity Structure

SeaORM generates the following components for each entity:

#### Model (Immutable Row Representation)
- **Purpose**: Represents a single row in the database table
- **Attributes**:
  - `#[sea_orm(table_name = "table_name")]`
  - `#[sea_orm(primary_key)]` on primary key fields
  - `#[sea_orm(column_name = "custom_name")]` for column name override
  - `#[sea_orm(column_type = "Text")]` for column type specification
  - `#[sea_orm(default_value = "value")]` for default values
  - `#[sea_orm(unique)]` for unique constraints
  - `#[sea_orm(indexed)]` for indexed columns
  - `#[sea_orm(nullable)]` for nullable fields
- **Generated**: Via `DeriveEntityModel` macro
- **Lifeguard Equivalent**: `LifeModel` (Story 01)

#### Column Enum
- **Purpose**: Enum listing all columns in the table
- **Usage**: Type-safe column references in queries (`Column::Name`)
- **Generated**: Via `DeriveEntityModel` macro
- **Lifeguard Equivalent**: Generated by `LifeModel` macro (Story 01)

#### PrimaryKey Enum
- **Purpose**: Enum defining primary key(s) of the table
- **Usage**: Composite primary keys, primary key operations
- **Generated**: Via `DeriveEntityModel` macro
- **Lifeguard Equivalent**: Generated by `LifeModel` macro (Story 01)

#### Entity Type
- **Purpose**: Type representing the entity itself
- **Usage**: `Entity::find()`, `Entity::find_by_id()`, etc.
- **Generated**: Via `DeriveEntityModel` macro
- **Lifeguard Equivalent**: Generated by `LifeModel` macro (Story 01)

#### ActiveModel (Mutable Change-Set)
- **Purpose**: Mutable version for inserts and updates
- **Features**:
  - Tracks changed fields (dirty tracking)
  - Can be created from Model
  - Can be converted to Model
  - Supports partial updates (only changed fields)
- **Generated**: Via `DeriveEntityModel` macro
- **Lifeguard Equivalent**: `LifeRecord` (Story 02)

#### Relation Enum
- **Purpose**: Enum specifying relationships with other entities
- **Types**:
  - `has_one`: One-to-one relationship
  - `has_many`: One-to-many relationship
  - `belongs_to`: Many-to-one relationship
  - `many_to_many`: Many-to-many relationship
- **Attributes**:
  - `from`: Foreign key column in this entity
  - `to`: Primary key column in related entity
  - `on_update`: Cascade behavior on update
  - `on_delete`: Cascade behavior on delete
- **Generated**: Via `DeriveRelation` macro
- **Lifeguard Equivalent**: Planned for Epic 05 (Story 06)

### 2. Derive Macros

#### DeriveEntityModel
- **Generates**: Model struct, Column enum, PrimaryKey enum, Entity type
- **Lifeguard Equivalent**: `#[derive(LifeModel)]` (Story 01)

#### DeriveRelation
- **Generates**: Relation enum with relationship definitions
- **Lifeguard Equivalent**: Planned for Epic 05 (Story 06)

#### DeriveActiveModel
- **Generates**: ActiveModel struct with change tracking
- **Lifeguard Equivalent**: `#[derive(LifeRecord)]` (Story 02)

### 3. Traits

#### EntityTrait
- **Methods**:
  - `find()` - Query builder
  - `find_by_id()` - Find by primary key
  - `find_one()` - Find first matching record
  - `find_all()` - Find all records (deprecated, use `find().all()`)
- **Lifeguard Equivalent**: Methods on `LifeModel` (Story 03)

#### ActiveModelTrait
- **Methods**:
  - `insert()` - Insert new record
  - `update()` - Update existing record
  - `save()` - Insert or update (upsert)
  - `delete()` - Delete record
- **Lifeguard Equivalent**: Methods on `LifeRecord` (Story 03)

#### ColumnTrait
- **Purpose**: Trait for column operations
- **Methods**: Column metadata, type information
- **Lifeguard Equivalent**: Generated by `LifeModel` macro (Story 01)

#### Related Trait
- **Purpose**: Defines how entities relate to each other
- **Implementation**: `impl Related<OtherEntity> for Entity`
- **Lifeguard Equivalent**: Planned for Epic 05 (Story 06)

#### ActiveModelBehavior
- **Purpose**: Defines behavior for ActiveModel operations
- **Implementation**: `impl ActiveModelBehavior for ActiveModel {}`
- **Lifeguard Equivalent**: Behavior traits for `LifeRecord` (Story 02)

### 4. Query Methods

#### Find Operations
- `Entity::find()` - Returns query builder
- `Entity::find_by_id(id)` - Find by primary key
- `Entity::find_one(filter)` - Find first matching record
- `Entity::find_all()` - Find all records (deprecated)
- **Lifeguard Equivalent**: Story 03

#### Query Builder Methods
- `find().filter(condition)` - Add WHERE clause
- `find().order_by(column)` - Add ORDER BY clause
- `find().order_by_desc(column)` - Add ORDER BY DESC clause
- `find().limit(n)` - Add LIMIT clause
- `find().offset(n)` - Add OFFSET clause
- `find().all()` - Execute query and return all results
- `find().one()` - Execute query and return first result
- `find().paginate(pool, page_size)` - Paginate results
- `find().count()` - Count matching records
- **Lifeguard Equivalent**: Story 05

#### Insert Operations
- `ActiveModel::insert(pool)` - Insert new record
- `ActiveModel::insert_many(models, pool)` - Batch insert
- `Entity::insert_many(models, pool)` - Batch insert (static method)
- **Lifeguard Equivalent**: Story 03 (batch operations in future)

#### Update Operations
- `ActiveModel::update(pool)` - Update existing record
- `ActiveModel::save(pool)` - Insert or update (upsert)
- `Entity::update_many(filter, values, pool)` - Batch update
- **Lifeguard Equivalent**: Story 03 (batch operations in future)

#### Delete Operations
- `Model::delete(pool)` - Delete record
- `Entity::delete_many(filter, pool)` - Batch delete
- `Entity::delete_by_id(id, pool)` - Delete by primary key
- **Lifeguard Equivalent**: Story 03 (batch operations in future)

### 5. Query Builder Features

#### Filtering
- `eq(value)` - Equal to
- `ne(value)` - Not equal to
- `gt(value)` - Greater than
- `gte(value)` - Greater than or equal
- `lt(value)` - Less than
- `lte(value)` - Less than or equal
- `like(pattern)` - LIKE pattern matching
- `ilike(pattern)` - Case-insensitive LIKE
- `is_in(values)` - IN clause
- `is_not_in(values)` - NOT IN clause
- `is_null()` - IS NULL
- `is_not_null()` - IS NOT NULL
- `between(start, end)` - BETWEEN clause
- `contains(value)` - Contains (for arrays/JSONB)
- `and(condition)` - AND condition
- `or(condition)` - OR condition
- **Lifeguard Equivalent**: Story 05

#### Ordering
- `order_by(column)` - ORDER BY ASC
- `order_by_desc(column)` - ORDER BY DESC
- `order_by_asc(column)` - ORDER BY ASC (explicit)
- Multiple ordering: `order_by(col1).order_by_desc(col2)`
- **Lifeguard Equivalent**: Story 05

#### Pagination
- `limit(n)` - LIMIT clause
- `offset(n)` - OFFSET clause
- `paginate(pool, page_size)` - Pagination helper
- `paginate_and_count(pool, page_size)` - Pagination with count
- **Lifeguard Equivalent**: Story 05

#### Aggregation
- `count()` - COUNT(*)
- `sum(column)` - SUM(column)
- `avg(column)` - AVG(column)
- `max(column)` - MAX(column)
- `min(column)` - MIN(column)
- **Lifeguard Equivalent**: Story 05

#### Joins
- `join(JoinType, RelatedEntity)` - JOIN related entity
- `left_join(RelatedEntity)` - LEFT JOIN
- `right_join(RelatedEntity)` - RIGHT JOIN
- `inner_join(RelatedEntity)` - INNER JOIN
- `join_rev(JoinType, RelatedEntity)` - Reverse join
- **Lifeguard Equivalent**: Planned for Epic 05 (Story 06)

#### Group By / Having
- `group_by(column)` - GROUP BY clause
- `having(condition)` - HAVING clause
- **Lifeguard Equivalent**: Story 05 (advanced features)

- `#[sea_orm(primary_key)]` - Primary key
- `#[sea_orm(column_name = "custom_name")]` - Custom column name
- `#[sea_orm(column_type = "Text")]` - Column type
- `#[sea_orm(default_value = "value")]` - Default value
- `#[sea_orm(unique)]` - Unique constraint
- `#[sea_orm(indexed)]` - Indexed column
- `#[sea_orm(nullable)]` - Nullable field
- `#[sea_orm(auto_increment)]` - Auto-increment
- `#[sea_orm(enum_name = "EnumName")]` - Enum type
- **Lifeguard Equivalent**: Story 01

### 7. Relation Types

- `has_one` - One-to-one relationship
- `has_many` - One-to-many relationship
- `belongs_to` - Many-to-one relationship
- `many_to_many` - Many-to-many relationship (via join table)
- **Lifeguard Equivalent**: Planned for Epic 05 (Story 06)

### 8. Additional Features

#### Transactions
- `pool.begin()` - Start transaction
- `transaction.commit()` - Commit transaction
- `transaction.rollback()` - Rollback transaction
- **Lifeguard Equivalent**: Planned for Epic 01 (future story)

#### Raw SQL
- `Entity::find_by_statement(statement)` - Execute raw SQL
- `Entity::execute_unprepared(sql)` - Execute unprepared SQL
- **Lifeguard Equivalent**: Via LifeExecutor (Epic 01)

#### Connection Pool
- `DatabaseConnection` - Connection pool type
- `pool.get_connection()` - Get connection from pool
- **Lifeguard Equivalent**: `LifeguardPool` (Epic 01, Story 04)

#### Migrations
- `MigrationTrait` - Trait for defining migrations with `up()` and `down()` methods
- `Migrator::up()` - Apply pending migrations
- `Migrator::down()` - Rollback migrations
- `Migrator::refresh()` - Rollback all and reapply
- `Migrator::reset()` - Rollback all migrations
- `Migrator::status()` - Check migration status
- `sea-orm-cli migrate` - CLI tool for migration management
- `sea-orm-cli migrate init` - Initialize migration directory
- `sea-orm-cli migrate create <name>` - Create new migration
- `sea-orm-cli migrate up` - Apply migrations
- `sea-orm-cli migrate down` - Rollback migrations
- `sea-orm-cli migrate status` - Show migration status
- Programmatic migrations (Rust code, not SQL files)
- Data seeding in migrations
- Atomic migrations (transactions in PostgreSQL)
- Conditional operations (`has_column`, `has_table`, etc.)
- **Lifeguard Equivalent**: Epic 03 (Migrations)
