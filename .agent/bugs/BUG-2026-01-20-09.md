# BUG-2026-01-20-09: Incorrect File Count Assertion in test_handle_generate_fails_when_file_exists

**Date:** 2026-01-20  
**Status:** ✅ FIXED  
**Priority:** High  
**Severity:** Test correctness - Test would pass even when second file isn't created

## Summary

The test `test_handle_generate_fails_when_file_exists` has a comment stating "Should have 2 files now (original + new)" when the operation succeeds (different timestamp), but the assertion only checks `assert!(files.len() >= 1, ...)`. This means the test would incorrectly pass even if the second file wasn't created, defeating the purpose of verifying atomic file creation behavior.

## Discovery

**Date:** 2026-01-20  
**Source:** Code review - assertion doesn't match comment  
**Severity:** `high`  
**Status:** `fixed`

## Location

- **File:** `lifeguard-migrate/src/main.rs`
- **Lines:** 711-712

## Description

In the test `test_handle_generate_fails_when_file_exists`, when `handle_generate()` succeeds (because the timestamp differs from the manually created file), the test should verify that both files exist:
1. The original file created manually at line 687
2. The new file created by `handle_generate()` with a different timestamp

However, the assertion at line 712 only checks `files.len() >= 1`, which would pass even if only 1 file exists. This means:
- If `handle_generate()` fails to create the new file, the test would still pass
- The test doesn't actually verify that atomic file creation works correctly
- The comment at line 711 is misleading

**Current code:**
```rust
// Should have 2 files now (original + new)
assert!(files.len() >= 1, "Should have at least the original file");
```

**Expected behavior:**
```rust
// Should have 2 files now (original + new)
assert_eq!(files.len(), 2, "Should have exactly 2 files (original + new)");
```

## Root Cause

The assertion was written too leniently, checking for "at least 1 file" instead of verifying the expected behavior of having 2 files when the operation succeeds.

## Fix

**Status:** ✅ FIXED

**Solution:** Changed the assertion from `assert!(files.len() >= 1, ...)` to `assert_eq!(files.len(), 2, ...)` to match the comment and properly verify that both files exist when the operation succeeds.

**Code change:**
```rust
// Should have 2 files now (original + new)
assert_eq!(files.len(), 2, "Should have exactly 2 files (original + new)");
```
