# BUG-2026-01-21-01: Migration Template Method Signatures Don't Match Migration Trait

**Date:** 2026-01-21  
**Status:** ✅ FIXED  
**Priority:** Critical  
**Severity:** Compilation error - Generated migration files fail to compile

## Summary

The migration template generated by `handle_generate` produces method signatures that don't match the `Migration` trait. Both `up` and `down` methods are generated with `&SchemaManager` (missing lifetime parameter) but the trait requires `&SchemaManager<'_>` with the lifetime parameter. Any migration file created using `lifeguard-migrate generate` will fail to compile due to trait method signature mismatch.

## Discovery

**Date:** 2026-01-21  
**Source:** Code review - template generation doesn't match trait definition  
**Severity:** `critical`  
**Status:** `fixed`

## Location

- **File:** `lifeguard-migrate/src/main.rs`
- **Lines:** 355, 366

## Description

The `Migration` trait (defined in `src/migration/migration.rs`) requires:
- `fn up(&self, manager: &SchemaManager<'_>) -> Result<(), LifeError>;`
- `fn down(&self, manager: &SchemaManager<'_>) -> Result<(), LifeError>;`

However, the template generated by `handle_generate()` in `lifeguard-migrate/src/main.rs` produces:
- `fn up(&self, manager: &SchemaManager) -> Result<(), LifeError> {`
- `fn down(&self, manager: &SchemaManager) -> Result<(), LifeError> {`

The generated code is missing the lifetime parameter `'_` in both method signatures, causing a compilation error when users try to compile their generated migration files.

**Error message users would see:**
```
error[E0053]: method `up` has an incompatible type for trait `Migration`
  --> migrations/.../20260121174149_example.rs:15:5
   |
15 |     fn up(&self, manager: &SchemaManager) -> Result<(), LifeError> {
   |     ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
   |
   = note: expected signature `fn up(&Self, &SchemaManager<'_>) -> Result<(), LifeError>`
              found signature `fn up(&Self, &SchemaManager) -> Result<(), LifeError>`
```

## Root Cause

The template string in `handle_generate()` was written without the lifetime parameter `'_` that is required by the `SchemaManager` type. This is a simple oversight in the template generation code.

## Fix

**Status:** ✅ FIXED

**Solution:** Updated the template to include the lifetime parameter `'_` in both `up` and `down` method signatures to match the `Migration` trait definition.

**Code change:**
```rust
// Before (lines 355, 366):
fn up(&self, manager: &SchemaManager) -> Result<(), LifeError> {{
fn down(&self, manager: &SchemaManager) -> Result<(), LifeError> {{

// After:
fn up(&self, manager: &SchemaManager<'_>) -> Result<(), LifeError> {{
fn down(&self, manager: &SchemaManager<'_>) -> Result<(), LifeError> {{
```
