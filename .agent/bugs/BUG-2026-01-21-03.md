## Summary

The variable `needs_db` is declared and assigned on line 111 of `lifeguard-migrate/src/main.rs` but is never used. The subsequent `match` statement on line 114 handles commands individually without referencing this variable, making it dead code.

## Discovery

**Date:** 2026-01-21

- **File:** `lifeguard-migrate/src/main.rs`

## Description

The code declares `needs_db` to determine which commands require a database connection:

```rust
let needs_db = !matches!(cli.command, Commands::GenerateFromEntities { .. } | Commands::Generate { .. });
```

However, this variable is never referenced. The code instead handles database connection requirements through the match statement structure:
- Commands that don't need a DB (`GenerateFromEntities`, `Generate`) are handled in specific match arms
- All other commands are handled in the `_` branch which connects to the database

The variable was likely intended to be used for early validation of database URL requirements, but this was never implemented.

## Root Cause

The variable was created with the intention of validating database URL requirements early (before entering the match statement), but the implementation was never completed. The code works correctly without it, but it represents dead code that should either be used or removed.

## Fix

Use the `needs_db` variable to validate database URL early for commands that require it. This provides:
1. Better error messages (fails fast before processing)
2. Clearer code intent
3. Removes dead code warning

The fix validates the database URL immediately after determining if it's needed, before entering the match statement.

No test changes needed - this is a code cleanup that improves error handling but doesn't change functionality.

## Impact

- **Code Quality:** Removes dead code warning
- **User Experience:** Better error messages (fails fast if DB URL missing)
- **Functionality:** No change - code works identically, just validates earlier
