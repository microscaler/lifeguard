# BUG-2026-01-21-02: Unused _applied_versions Variable in Migrator::status

**Date:** 2026-01-21  
**Status:** ✅ FIXED  
**Priority:** Low  
**Severity:** Code quality - Dead code

## Summary

The variable `_applied_versions` is created as a `HashSet<i64>` from applied migrations but is never used. The code later uses `file_versions` for checking missing files and `applied.iter().find()` for checking if migrations are applied, making `_applied_versions` completely redundant dead code.

## Discovery

**Date:** 2026-01-21  
**Source:** Code review / User report  
**Severity:** `low`  
**Status:** `fixed`

## Location

- **File:** `src/migration/migrator.rs`
- **Lines:** 59-62 (removed)

## Description

In the `Migrator::status()` method, a `HashSet<i64>` called `_applied_versions` is created from the applied migrations (lines 60-62), but it is never actually used in the code. The method instead:

1. Uses `applied.iter().find(|r| r.version == file.version)` on line 74 to check if a migration file is applied
2. Uses `file_versions.contains(&record.version)` on line 98 to check for missing files

The `_applied_versions` HashSet was likely intended to optimize lookups but was never integrated into the logic.

## Root Cause

The variable was probably created with the intention of optimizing lookups (O(1) HashSet lookup vs O(n) linear search), but the implementation was never completed. The code works correctly without it, but it represents dead code that should be removed.

## Fix

Removed the unused `_applied_versions` variable and its associated comment. The code continues to work correctly using:
- `applied.iter().find()` for finding applied migration records (needed for checksum validation)
- `file_versions.contains()` for checking if migration files exist

No test changes needed - this is a code cleanup that doesn't affect functionality.
